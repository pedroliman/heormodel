# 4 — Documentation website with quartodoc

**Goal:** a public documentation site built with [Quarto](https://quarto.org)
and [quartodoc](https://machow.github.io/quartodoc/) (API reference
generated from the package's docstrings), published from CI.

This can proceed in parallel with items 1–3; the API reference regenerates
as engines land.

## Structure

```
docs/
  _quarto.yml          # site config: navbar, theme (light + dark), search
  index.qmd            # landing page: the core idea + quickstart (from README)
  get-started.qmd      # install (uv), bring-your-own-outputs in 10 lines
  tutorials/
    byo-outputs.qmd        # narrated version of examples/byoo_example.py
    full-pipeline.qmd      # params -> run_psa -> cea -> voi
    calibration.qmd        # narrated version of the item-01 example
  concepts/
    architecture.qmd       # the outcome schema contract, shared iteration index
    engines.qmd            # engine protocol; what "contract on outputs" means
  reference/           # generated by quartodoc — do not hand-edit
```

## quartodoc configuration

- `quartodoc:` section in `_quarto.yml` with `package: heval`, one
  reference section per subpackage (`params`, `models`, `run`, `cea`,
  `voi`, `calibrate`, `report`), listing the public names from each
  `__init__.py` `__all__`.
- Docstrings are already Google-style with worked `Example:` blocks —
  quartodoc renders these directly; the doctest suite
  (`uv run pytest --doctest-modules src/heval`) keeps every example on the
  site executable and true.
- Interlinks to numpy/pandas/scipy docs via the quartodoc interlinks
  filter.

## Tutorials are executed, not pasted

`.qmd` tutorials execute at render time against the installed package, so
the site fails to build if the API drifts — treat the docs build as a test.
Keep runtimes small (thousands of PSA iterations, not the 80k used in the
validation tests).

## Tooling and CI

- Dev extra: add `quartodoc` to `[dev]` (Quarto itself is a system binary,
  installed in CI via `quarto-dev/quarto-actions/setup`).
- Local workflow: `uv run quartodoc build --config docs/_quarto.yml` then
  `quarto preview docs`.
- GitHub Actions workflow `docs.yml`: on push to `main`, build and deploy
  to GitHub Pages (`quarto publish gh-pages` or the Pages deploy action).
- Add the site URL to `pyproject.toml` `[project.urls]` and the README.

## Acceptance

- Site builds clean in CI from a fresh checkout.
- Every public symbol exported by the subpackage `__init__.py` files
  appears in the reference.
- The three tutorials execute during the build and their outputs render.
